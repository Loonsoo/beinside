# BeInside 콘텐츠 작성 스킬

## 작업 전 필수 확인

콘텐츠 작업을 시작하기 전에 아래 문서를 반드시 읽으세요.

| 문서 | 읽어야 하는 상황 |
|------|-----------------|
| `references/tone-guide.md` | 모든 콘텐츠 작성 시 (항상) |
| `references/data-schema.md` | js/data.js에 새 데이터를 추가하거나 기존 구조를 수정할 때 |
| `references/source-list.md` | 통계·수치를 콘텐츠에 포함할 때 |
| `references/persona-summary.md` | 타겟 대상을 정의하거나 톤을 결정할 때 |

민감한 콘텐츠(가정폭력·자해·위기 상황)가 포함되면 **beinside-safety 스킬**을 함께 사용하세요.
UI 구현이 필요하면 **beinside-ui 스킬**을 함께 사용하세요.
다국어 콘텐츠라면 **beinside-i18n 스킬**을 함께 사용하세요.

## 톤 규칙 (3가지 모드)

### 모드 1: 기본 톤 (부모·일반 성인)
- **허용**: "~해 보세요", "~할 수 있어요", "~됩니다", "~예요"
- **금지**: "~해야 합니다", "~하십시오", "~해야만 해요"
- 부드럽고 제안하는 형식. 지시가 아니라 동반자처럼.

### 모드 2: 청소년 톤
- **허용**: 반말 또는 부드러운 해요체. "~거든", "~야", "~인 거야", "~해봐", "~괜찮아"
- **필수**: 문장 짧게. 한 문단 최대 2줄.
- **필수**: 이모지 적극 사용 (💛 ✨ 🌱 👋 등)
- **금지**: "부모님" → 대신 "어른", "어른들" 사용
- **금지**: 설교하는 톤, 정보 나열식 구조

### 모드 3: 통계 톤
- **형식**: 수치 먼저 → 바로 아래에 "이게 무슨 의미냐면..." 한 줄 해석 필수
- 예: "27.7% — 이게 무슨 의미냐면, 교실에서 세 명 중 한 명이 우울감을 느끼고 있다는 거예요."

### 공통 금지 표현
- 판단적 표현: "잘못해서", "나쁜 부모", "이렇게 하면 안 되는데", "실수한 거예요"
- 명령형: "~해야 합니다", "~하십시오", "~해야 해요"
- 의학적 단정: "~이 확실합니다", "~하면 반드시 ~됩니다"

## 2단계 정보 구조 (모든 콘텐츠 필수)

모든 콘텐츠는 다음 구조를 따릅니다.

```
1단계 (항상 보임)
├── 핵심 3줄 요약
└── "이것만은 꼭" 경고 1개 (빨간 톤 강조)

2단계 (접혀 있음)
└── 상세 카드 → 아코디언 또는 탭으로 접힌 상태
```

- **모바일 기준** 초기 로딩 시 3스크롤 이내에만 1단계가 보여야 함
- 2단계 카드는 기본 `hidden` 또는 `collapsed` 상태

## 대상 뱃지 규칙

모든 카드에 다음 뱃지 중 하나 이상을 반드시 표시하세요.

| 뱃지 | 의미 |
|------|------|
| 👨‍👩‍👧 부모·보호자 | 부모 또는 보호자가 읽어야 하는 내용 |
| 🧒 청소년 본인 | 청소년이 직접 읽는 콘텐츠 |
| 🙋 본인(성인) | 성인 본인이 읽는 콘텐츠 |
| 👀 모두가 읽으면 좋아요 | 전 연령 공통 |

## 통계 출처 규칙

1. 모든 수치에 출처를 명시합니다. 출처 없는 통계는 사용하지 않습니다.
2. 사용 전 `references/source-list.md`에서 사용 가능한 출처를 확인합니다.
3. 새 출처를 사용할 경우:
   - `references/source-list.md`에 항목 추가
   - `index.html`의 출처 드로어(`#source-drawer`) HTML에도 반영

## 긴급 연락처 작성 규칙

긴급 연락처를 포함할 때 각 번호에 다음을 반드시 작성합니다.

```
전화번호: [번호]
이름: [기관명]
설명: [무료 여부] / [24시간 여부] / [전화하면 어떻게 되는지 한 줄]
비고: "자세한 내용(비밀보장 범위, 기록 정책)은 전화 시 확인해 주세요."
```

- 비밀보장·기록 정책을 단정 짓지 않습니다. 기관마다 다를 수 있습니다.
- 전화번호는 `<a href="tel:1393">` 형태로 항상 tel: 링크를 달아야 합니다.

## 콘텐츠 완료 후 자동 검수 체크리스트

콘텐츠 작성이 완료되면 아래를 순서대로 확인합니다.

### ✅ 톤 검증
- [ ] 기본 톤: "~해야 합니다", "~하십시오" 없는지 확인
- [ ] 청소년 콘텐츠: 문단 2줄 이내, 이모지 사용, "부모님" → "어른" 교체 확인
- [ ] 통계: "이게 무슨 의미냐면..." 해석 문장 포함 확인
- [ ] 판단적 표현 없는지 전체 검색

### ✅ 구조 검증
- [ ] 1단계(3줄 요약 + 경고) 항상 보이는지 확인
- [ ] 2단계 카드 접힌 상태로 시작하는지 확인
- [ ] 모바일 3스크롤 이내에 핵심 정보 완료되는지 확인

### ✅ 출처 검증
- [ ] 모든 통계 수치에 출처 명시 확인
- [ ] 새 출처 사용 시 source-list.md + 출처 드로어 업데이트 확인

### ✅ 뱃지 검증
- [ ] 모든 카드에 대상 뱃지(👨‍👩‍👧 🧒 🙋 👀) 포함 확인

### ✅ 안전 검증 (민감 주제일 경우)
- [ ] beinside-safety 스킬 확인 여부
- [ ] 긴급 연락처 포함 확인
- [ ] 피해자 탓 표현 없는지 확인

## 데이터 추가 방법

`js/data.js`에 데이터를 추가할 때 `references/data-schema.md`의 구조를 정확히 따르세요.
새 콘텐츠의 기본 템플릿은 `assets/content-template.json`을 복사하여 시작합니다.
